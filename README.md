# Logisimulation

物流をシミュレーションするゲームプラットフォームです。

## ゲーム概要

日本の主要都市間でトラックを使って貨物を配送する物流シミュレーションゲームです。

### 特徴

- 🗾 **10都市マップ**: 東京・大阪・名古屋・札幌・福岡・仙台・広島・金沢・新潟・松山
- 🚛 **トラック管理**: 複数のトラックを所有・管理
- 📦 **注文システム**: ランダムに発生する配送注文
- 💰 **経済システム**: 配送報酬・維持費・トラック購入
- 🗺️ **経路探索**: ダイクストラ法による最短経路自動計算
- ⏱ **ゲーム速度**: 1×〜8×の速度調整

### 操作方法

1. **トラックを選択**: サイドパネルの「トラック」タブでトラックをクリック
2. **注文を割り当て**: 「注文」タブで注文をクリックして配送を開始
3. **トラックを購入**: 資金が ¥500,000 以上の時にトラックを追加購入可能
4. **速度調整**: 画面上部で 1×〜8× の速度でシミュレーションを進める

## 開発・実行

```bash
npm install
npm run dev      # 開発サーバー起動
npm run build    # プロダクションビルド
npm run preview  # ビルドのプレビュー
```

### Windows 初回セットアップ

このプロジェクトの実行には **Node.js (LTS)** が必要です。`npm` コマンドが見つからない場合は、以下を実行してください。

```powershell
winget install OpenJS.NodeJS.LTS --accept-package-agreements --accept-source-agreements
```

インストール直後は、既存のターミナルでは `node` / `npm` が認識されない場合があります。
その場合は **VS Code のターミナルを開き直す** か、VS Code を再起動してから再実行してください。

## 資源・輸送システム（v2）

問題定義に基づく「資源の輸送と消費」機能の追加。

### 都市の資源ストック

各都市（Node）は以下を保持します:

```json
{ "population": 0, "food": 0, "iron": 0, "book": 0, "monster_parts": 0 }
```

### 輸送命令（荷馬車）

```json
{ "from": "CityA", "to": "CityB", "items": { "food": 100, "iron": 50 }, "unit": "wagon" }
```

### エンジンロジック

1. **食料消費** — 毎50ターンごとに人口に応じた食料を消費。食料不足時は人口が減少。
2. **輸送ロス** — 地形係数（山岳=0.5, 平地=0.8, 街道=0.9）をかけた資源が目的地に到着。
3. **腐敗値（Corruption）** — 輸送中に失われた資源が腐敗値として累積。500を超えるとカオスイベント発生。

### 操作方法（資源・輸送タブ）

- 🏙 **[資源]タブ** — 全都市の資源ストックと人口を確認し、上部のデモボタンから「拠点A(札幌)→拠点B(仙台)へ食料120輸送」をワンクリック実行
- 🐴 **[輸送]タブ** — 出発地・目的地を選択し、各資源の輸送量を入力して「輸送命令を発行」
- ⏱ **ターン経過の観察** — 資源タブで次の食料消費までの残り tick と腐敗値の増加を追跡
